서버 제출 테스트1

  	이름과 나이를 입력받아서 서버(Java 코드)로 제출
  	
  	* 준비물 *
  	
  	1) form 태그 : 내부에 작성된 input 값을 지정된 경로로 제출
  	- action : 제출할 경로
  	- method : 제출 방법(GET/POST) 기본값 : GET
  	
  	제출 == 클라이언트가 서버로 요청
  	
  	2) input 태그 : 값을 입력하는 양식을 제공
  	- name (*중요*) : 서버로 제출 시 key 역할
	  - value or 입력된 값 : 서버로 제출시 value 역할
	
	
	  ?inputName=홍길동
	
    ** GET : GET을 통한 요청은 URL 주소 끝에 파라미터로 포함되어 전송되며,
    이 부분을 query String 이라고 부른다.
    방식은 URL 끝에 "?" 를 붙이고 그 다음 변수명1=값1, 여러개면 변수명1=값1&변수명2=값2...
    ex) www.example.com/?name1=value1&name2=value2...
    -> 서버에서는 name1과 name2라는 파라미터 명으로 
      각각 value1과 value2 값을 전달받을 수 있다.

     [장점]
     - 단순함
     - 캐싱 가능 (주소 저장 == 북마크, 즐겨찾기, 주소 공유)

     [단점]
     - 보안에 취약하다(요청데이터가 주소에 그대로 노출됨)
     - 요청데이터의 길이 제한이 있다.
